# TravelInfoShare 旅游信息分享平台
一款轻量、高性能的跨平台旅游信息分享平台，整合**旅游攻略分享、社交互动、行程规划**三大核心能力，为旅行者打造攻略收集、队伍组建、实时沟通、路线规划的一站式解决方案，全面覆盖旅游前、中、后全流程的服务需求。

## 项目概述
随着旅游业的数字化发展，旅行者对一体化、互动化的旅游信息服务需求日益提升，而传统旅游信息获取方式存在**内容零散、更新滞后、缺乏社交属性**等问题，无法满足用户从出行规划、途中互动到后期分享的全流程需求。

本项目基于Qt框架与MySQL数据库打造，采用分层架构设计，实现了旅游信息服务的全闭环。平台以数据高效存储与交互为核心，兼顾功能完整性与操作易用性，为旅游爱好者打造一个可交流、可协作、可分享的个性化旅游信息社区，同时具备良好的扩展性与跨平台适配性。

## 核心功能
### 1. 用户身份认证
- 支持账号唯一校验的注册/登录/退出功能，密码加密存储保障安全
- 登录状态实时同步，账号状态管理（正常/禁用），支持权限基础控制
- 前端+后端双重输入校验，杜绝非法数据提交

### 2. 旅游攻略服务
- **热门攻略推荐**：首页按热度值降序展示热门攻略，快速获取优质旅游内容
- **攻略发布/浏览**：支持发布带**城市、分类标签**的攻略，详情页完整展示攻略内容
- **多维度筛选**：按城市（如北京、桂林）、分类（自然风景/人文景点等）精准筛选攻略
- **收藏管理**：攻略收藏/取消收藏，个人中心可集中查看、删除收藏的攻略

### 3. 社交互动功能
- 好友管理：添加/删除好友、查看好友列表，做**自加好友、重复添加**的防错限制
- 实时聊天：与好友一对一即时通讯，聊天记录持久化存储，历史消息随时可查
- 消息同步：发送消息后自动刷新聊天记录，前端展示与数据库数据实时一致

### 4. 个人中心管理
- 个人信息管理：查看/编辑昵称、个性签名，修改后实时同步至数据库
- 安全改密：修改密码需验证原密码，从底层保障账号密码安全
- 收藏汇总：集中展示所有收藏攻略，支持一键删除，操作反馈清晰
- 信息可视化：直观展示账号、昵称、注册时间等核心个人信息

### 5. 行程规划与管理
- **自定义路线创建**：支持创建专属旅行路线，可填写详细的每日行程、景点安排等内容
- **旅行队伍组建**：添加/移除队员、查看队员列表，方便结伴出行的团队规划
- **数据持久化**：路线与队员信息永久存储，可随时查询、编辑，满足旅游途中的路线备份需求

## 核心技术亮点
### 1. 安全的数据处理机制
- 密码采用**MD5加密**存储，数据库中无明文密码，从根源避免密码泄露
- 所有SQL操作均使用**参数绑定**方式，彻底杜绝SQL注入攻击
- 前端输入校验+后端二次校验，确保入库数据的合法性与规范性
- 核心字段设置**唯一索引、外键约束**，避免重复数据、无效数据插入

### 2. 高优化的数据库设计
- 基于业务场景设计7张核心业务表，表结构合理，数据关系清晰，贴合实际业务需求
- 针对高频查询字段（username、user_id、city、category等）建立**定向索引**，大幅提升查询效率
- 原子性操作采用**事务管理**，确保多步操作的一致性（如添加好友、攻略收藏）
- 外键约束配置`ON DELETE CASCADE`，实现关联数据自动清理，减少冗余数据
- 多表关联查询使用`JOIN`语句替代多次单表查询，减少数据库交互开销

### 3. 无缝的前后端集成
- 基于Qt**信号与槽机制**，实现QML（前端）与C++（后端）的异步通信，交互流畅无卡顿
- `SqlApi`类采用**单例模式**设计，统一管理所有数据库操作，减少资源浪费，提升代码复用性
- 数据修改后**自动刷新相关列表**（如好友列表、收藏列表），实现前端展示与数据库的实时同步
- 封装可复用QML组件（聊天面板、筛选组件、攻略列表），减少代码冗余，便于维护

### 4. 稳定高效的性能表现
- 优化数据库连接池，增加**旧连接清理逻辑**，避免程序重启后的连接冲突
- 数据库单次查询/更新响应时间≤500ms，核心数据首次加载≤1s，交互体验流畅
- 支持100+用户同时在线操作，无数据冲突，满足小规模并发使用需求
- 对攻略列表、聊天记录等大数据集采用**分页加载**，优化页面加载性能

## 环境配置
### 前置条件
1. 操作系统：Windows 10/11（64位）、Linux（Ubuntu 20.04及以上）
2. 安装Qt 6.8.0（需包含**Qt Quick、Qt SQL、MySQL驱动**组件）
3. 安装MySQL 8.0.36（如需远程连接，请开启对应权限）
4. 安装CMake 3.16+及C++编译器（MSVC 2019+/GCC 9+）
5. （可选）安装Navicat Premium 16，方便数据库可视化管理

### 数据库配置
1. 新建MySQL数据库`travelInfo`，执行以下SQL语句：
```sql
CREATE DATABASE IF NOT EXISTS travelInfo DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE travelInfo;
```
2. 执行项目`/sql`目录下的SQL脚本，自动创建核心业务表与索引
3. 根据本地MySQL配置，修改`SqlApi.cpp`中的数据库连接参数：
```cpp
m_db.setHostName("127.0.0.1");    // 本地数据库地址
m_db.setPort(3306);               // MySQL默认端口
m_db.setUserName("你的MySQL用户名");
m_db.setPassword("你的MySQL密码");
```

## 项目结构
项目采用**模块化设计**，前端（QML）与后端（C++）代码完全分离，职责清晰，便于维护与扩展，核心目录结构如下：
```plain
TravelInfoShare/
├── qml/                  # 前端QML界面文件
│   ├── LoginPage.qml     # 登录/注册页面
│   ├── MainPage.qml      # 首页（热门攻略推荐）
│   ├── NewsPage.qml      # 社交聊天/好友管理页面
│   ├── CommunityPage.qml # 攻略发布/筛选页面
│   ├── TripPage.qml      # 行程路线/队伍管理页面
│   ├── SelfPage.qml      # 个人中心页面
│   └── Components/       # 可复用QML组件（聊天面板、筛选组件等）
├── src/                  # 后端C++源代码
│   ├── main.cpp          # 项目入口文件
│   ├── SqlApi.h/cpp      # 核心数据库操作类（单例模式）
│   ├── models/           # 数据模型类（攻略信息、用户信息等）
│   └── utils/            # 工具类（MD5加密、数据校验等）
├── sql/                  # 数据库初始化脚本（建表+建索引）
├── CMakeLists.txt        # CMake跨平台构建配置文件
└── README.md             # 项目开源说明文档
```
